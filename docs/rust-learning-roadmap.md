# Rust 学习程度自检与进阶大纲

本文回答三件事：**基础是否算学完**、**如何验证掌握度**、**下一步怎么练**（含可持续节奏与分阶段大纲）。  
结合你当前仓库：已有 **p01～p19 系列练习、所有权/借用、测试、minigrep、智能指针、并发、宏与 unsafe** 等，**语法与书上的主线话题覆盖面已经很大**；是否「学完善」取决于你是否能在**脱离教程**的情况下独立写出来，见下文自检。

---

## 1. 「基础语法」是否已学完善？

### 1.1 客观说：覆盖 ≠ 内化

- **覆盖**：你的工程目录显示已触及 **The Rust Programming Language（官方书）** 大部分章节对应主题，**就「见过」而言非常充分**。
- **内化**：若仍存在「编译器报错要看很久」「不知道用 `&` 还是 `mut`」「`async` 项目不敢开」等情况，说明**熟练度与迁移能力**仍在进阶途中，这很正常。

结论：**不必再等「基础完美」才进阶**；宜采用 **「并行巩固 + 向前拓展」**：每周留固定时间复盘旧题，同时开新专题。

### 1.2 建议的「基础完备」底线（可自检）

下面各项应能在 **不翻书** 或 **少翻书** 下完成：

| 能力 | 自检标准 |
|------|-----------|
| 所有权与借用 | 能解释为何某处必须 clone / 用引用；能改通借用检查报错 |
| 错误处理 | 会写 `Result`，会在 `main` 里传播或 `match`；理解 `?` |
| 模块与可见性 | 会拆 `mod`、会 `pub use`，理解 crate 根 |
| 泛型与 trait | 能给结构体加 trait bound；会用 `impl Trait` 做参数 |
| 常用集合与迭代器 | 会用 `Iterator` 链式处理，避免多余 `clone` |
| 测试 | 会写单元测试、`#[should_panic]`、集成测试目录结构 |
| 并发基础 | 理解 `Send`/`Sync` 大意；会用 `Arc`、`Mutex`、channel 做简单任务 |

若多项仍吃力：**优先补** 所有权/借用 + `Result` + 模块，再推进 async / 大工程。

## 2. 如何验证学习程度（可量化）

### 2.1 短期自测（1～3 天）

1. **闭卷小题目**（本仓库或笔记本里抽题）  
   - 例如：实现「链表/缓存 LRU 简化版」「小型 CLI 读文件统计词频」「带错误的 JSON 解析封装」。  
   - 要求：**自己设计类型**，少用 `unwrap`，写完 `cargo clippy`。

2. **报错解释**  
   - 故意保留一次编译错误截图，**口头或写出来**：错误类别（move、生命周期、类型不匹配）及改法。能讲清即过关。

3. **Rustlings（强烈推荐）**  
   - 仓库：<https://github.com/rust-lang/rustlings>  
   - 按章节做小练习，**统计通过率**；错得多的章节对应你的薄弱点。

### 2.2 中期自测（1～2 周）

1. **独立 mini 项目（无分步答案）**  
   - 例如：HTTP 客户端拉 API + 本地缓存、简单 TUI、静态站点生成器一页版。  
   - 标准：**结构清晰**、`README` 写清如何运行、有基本错误处理。

2. **给别人讲 15 分钟**  
   - 主题任选：生命周期、Pin、或并发一种模型。讲不清的地方就是缺口。

### 2.3 长期指标（持续）

- 能稳定阅读 **中等体量开源 Rust 代码**（如小型 CLI 库），并改 PR 级别的小功能。  
- 遇到新 crate 能较快从 **docs.rs** 与示例上手。

## 3. 下一步进阶方向（总览）

按多数学习者路径，**在本书主线练习之后**常见进阶轴为：

| 阶段 | 核心主题 | 产出物 |
|------|-----------|--------|
| A. 巩固期 | 所有权、错误、迭代器、测试 | 3～5 个闭卷小练习 + Rustlings 高通过率 |
| B. 应用期 | **Cargo 工作区**、配置、日志、CLI/小服务 | 一个多 crate 或带配置的完整小项目 |
| C. 异步与生态 | **Tokio**、常见 async trait 模式、HTTP | 并发请求或简易 async Web/工具 |
| D. 深度可选 | **unsafe/FFI**、**宏**、性能与 **unsafe 审计**、嵌入式/WASM | 与职业方向绑定的专题项目 |

你仓库里 **p16 并发、p15 智能指针、p19 宏/unsafe** 已有「见过」基础；进阶重点是 **在真实项目里反复用**，而不是再堆同一难度的 demo。

## 4. 完整学习计划（大纲 + 节奏）

下面按 **约 16～20 周** 规划（可按你每周可用时间等比例伸缩）。默认：**每周至少 5～8 小时有效编码**，其中 **30% 复盘旧知识**。

### 阶段 0：对齐基线（1 周）

- 用 **§2.1** 做一次摸底；列出 **TOP 3 薄弱点**（如生命周期、线程、宏）。  
- 跑通 **`cargo fmt` / `clippy` / `test`** 作为所有练习的默认收尾。  
- 阅读本仓库 **docs/rust-engineering-practices.md**，给自己定 **最小规范**（fmt + clippy）。

### 阶段 1：内化基础（3～4 周）

**目标**：闭卷写小逻辑不卡壳。  

- **周 1～2**：Rustlings + 本仓库 **薄弱章节重练**（重写 `main`，不复制旧代码）。  
- **周 3**：自命题 **2 个** 200 行内小程序（CLI + 错误处理）。  
- **周 4**：**代码走读**：选一个 crates.io 上 star 不多的 CLI，读结构，画模块依赖草图。

**练习量建议**：每天至少 **45～90 分钟** 写代码；周末 **一次 2 小时** 整合小项目。

### 阶段 2：工程化小项目（3～4 周）

**目标**：体验「可交付」结构。  

- 用 **workspace** 拆 `lib` + `bin`（可参考 **docs/rust-engineering-practices.md §2**）。  
- 引入 **`tracing` 或 `log` + `env_logger`**、配置（env 或文件）。  
- 写一个 **完整 README** + **基本 CI 思路**（本地脚本即可，后续再接 GitHub Actions）。

**项目idea 示例**：文件监视 + 规则匹配工具、Markdown 链接检查、本地 JSON 合并 CLI。

### 阶段 3：异步与网络（4～5 周）

**目标**：建立 async 心智模型，敢写 I/O 密集程序。  

- 官方/async-book 选读 + **Tokio tutorial** 风格小例子。  
- 小项目：**并发拉取多个 URL**、带超时的客户端、或 **axum/warp** 极简 API。  
- 刻意练：**spawn**、**channel**、**取消/超时** 之一（不必一次全上）。

**注意**：async 与 **阶段 1 的同步并发**要分清；练习里写清「为何此处用 async」。

### 阶段 4：深度选修（持续，按方向二选一或多选）

- **系统/后端**：数据库客户端、连接池、迁移工具；读 **serde**、**sqlx/diesel** 其一。  
- **工具链**：写 **自定义 clippy lint** 或 **过程宏**（你已有 p19 基础，可做一个真实小需求）。  
- **FFI/unsafe**：在 **严格封装边界** 下做最小 C 互调；配合 **Miri**（若适用）与安全审查意识。  
- **前端/WASM**：**wasm-bindgen** 小 demo。

每选一个方向，以 **「一个可展示的仓库」** 结束该方向的第一轮。

### 复习节奏（全程嵌入）

- **每日**：开始前 **10 分钟** 复习昨天一个知识点（手写一句解释）。  
- **每周**：**半天** 重做一道 **两周前** 的题或错题。  
- **每月**：**一页纸** 总结本月新掌握的 5 个模式（如 `Cow`、`Pin`、某种并发模式）。
